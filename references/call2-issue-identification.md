# 调用 2：问题识别

## 执行规则

本文是调用 2 的完整指令。严格按本文操作。

**输入**：调用 1 的路由结果 + 合同全文
**输出**：结构化问题清单
**禁止输出**：示范文本比对、审批触发判定、谈判策略（调用 3 的事）

## 步骤

### 1. Playbook 按需加载

- 已命中 Playbook → 读取 `clauses` 章节的全部键名（约 1k tokens）
- 扫描合同条款，用 `clauseTypes` 关键词匹配对应的 clause 条目
- 只加载匹配的条目（一般 5-12 个，约 5-10k tokens）
- 未匹配 Playbook → 读取 `references/review-framework.md` 的通用风险清单

### 2. 项目 AGENTS.md 四维审核

对照项目的 CLAUDE.md 或 AGENTS.md 四维审核法，逐条确认：

| 维度 | 检查 |
|:---|:---|
| 业务风险 | 合同是否覆盖该类房地产业务的常见风险？ |
| 法律专业性 | 法言法语是否规范？权利义务是否清晰？免责限责是否有效？ |
| 条款一致性 | 付款-交付-违约条款是否自洽？正文与附件是否冲突？ |
| 业务需求还原度 | 关键商业安排是否写入？优惠条件是否固定？ |

### 3. 条款级比对

对每个匹配的 clause 条目，执行比对：

```
合同条款 → 读取 playbook.standard[] → 符合/偏离
                                      → 读取 playbook.acceptable[] → 可接受/不可接受
                                      → 读取 playbook.redFlags[] → 触发/未触发
```

每个偏离项取 playbook 的 `legalBasis`、`approvalTrigger`、`fallback` 字段。

### 4. 输出问题清单

每个问题按以下字段输出。**每个字段必须填写，不可省略**：

| 字段 | 说明 |
|:---|:---|
| 风险名称 | 简明描述 |
| 风险等级 | P0（签前必须）/ P1（优先谈判）/ P2（可优化） |
| 风险后果 | 如不修改可能导致的具体后果 |
| 判别标准 | 合同条款 vs playbook standard 的差异 |
| 推荐措辞 | 可直接落文的修改建议文本 |
| 法条依据 | `> 《X法》第X条："原文"` |
| 整改建议 | 修改类型：直接修订 / 局部删减 / 局部补入 / 整条重写 / 仅批注 |
| 相关条款 | 合同中的条款编号 |

### 5. 风险排序

输出时按以下顺序排列：效力与程序 → 资金与交付 → 救济与退出 → 文本优化

### 6. 汇总统计

问题清单末尾输出统计：

```
P0: X 项 | P1: X 项 | P2: X 项
触发调用 3 条件：P0≥3 [是/否] | 金额≥1亿 [是/否] | 上市公司 [是/否] | 跨境 [是/否] | 对赌 [是/否]
```

## 输出格式

```
## 结构化问题清单

### 🔴 P0（X 项）

**1. [风险名称]**
- 风险后果：[…]
- 判别标准：[合同条款] → [偏离 playbook.standard]
- 推荐措辞：[可直接落文的文本]
- 法条依据：> 《民法典》第X条：“原文”
- 整改建议：[直接修订 / 仅批注 / …]
- 相关条款：[第X条]

[... 更多 P0 项]

### 🟡 P1（X 项）
[... 同上格式 ...]

### 🟢 P2（X 项）
[... 同上格式 ...]

---

统计：P0 X 项 | P1 X 项 | P2 X 项
触发调用 3：[是/否]（如是，原因：[P0≥3 / 金额 / …]）
```

## 完成

调用 2 完成。输出问题清单。
**暂停。** 如触发调用 3 条件，询问用户："共 P0 X 项 / P1 X 项。是否进入深度分析（关键合同模式检查+审批触发+谈判策略）？[是/否]"

用户确认后再进入调用 3 或 Step 4。
